* Named Tuple
#+begin_export html
<p>
#+end_export

Typed version of [[https://docs.python.org/3/library/collections.html#collections.namedtuple][collections.namedtuple().]]

Usage:

#+BEGIN_SRC python :results silent :exports code
class Employee(NamedTuple):
    name: str
    id: int
#+END_SRC

This is equivalent to:

#+BEGIN_SRC python :results silent :exports code
Employee = collections.namedtuple('Employee', ['name', 'id'])
#+END_SRC

To give a field a default value, you can assign to it in the class body:

#+BEGIN_SRC python :results silent :exports code
class Employee(NamedTuple):
    name: str
    id: int = 3

employee = Employee('Guido')
assert employee.id == 3
#+END_SRC

Fields with a default value must come after any fields without a default.

The resulting class has an extra attribute __annotations__ giving a dict that maps the field names to the field types. (The field names are in the _fields attribute and the default values are in the _field_defaults attribute, both of which are part of the namedtuple()[fn:n-tup] API.)
